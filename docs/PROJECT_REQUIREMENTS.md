# Требования к проекту: Telegram Mini-App для продавцов

## Общее описание

Telegram Mini-App для управления продажами, складом, долгами и отчетами. Приложение предназначено для продавцов электронных сигарет и сопутствующих товаров.

---

## Технический стек

### Frontend
- **Framework:** Next.js 14+ (App Router)
- **UI Library:** shadcn/ui
- **Styling:** Tailwind CSS
- **State Management:** React Query (@tanstack/react-query)
- **Type Safety:** TypeScript
- **Icons:** Lucide React

### Backend
- **ORM:** TypeORM 0.3+
- **Database:** PostgreSQL 12+
- **Runtime:** Node.js 18+
- **API:** Next.js API Routes (Route Handlers)

### Интеграции
- **Telegram:** Telegram Mini App API, Telegram Bot API (Telegraf)
- **Authentication:** Telegram initData validation, custom accessKey

## База данных

### Основные сущности

#### User (Пользователь)
- `id` (UUID, PK)
- `telegramId` (string, unique)
- `username`, `firstName`, `lastName` (string | null)
- `role` ('seller' | 'client' | 'admin')
- `accessKey` (string, unique, nullable) - для входа без Telegram
- `subscriptionStatus` ('trial' | 'active' | 'expired')
- `trialEndsAt`, `subscriptionEndsAt` (Date | null)
- `referralCode` (string, unique, nullable)
- `referrerId` (UUID, FK → User.id, nullable)
- `isActive` (boolean)

#### Shop (Магазин)
- `id` (UUID, PK)
- `name` (string)
- `ownerId` (UUID, FK → User.id)
- `timezone` (string)
- `currency` (string)
- `address` (string | null)
- `supportTelegramUsername` (string | null)

#### Category (Категория)
- `id` (UUID, PK)
- `shopId` (UUID, FK → Shop.id)
- `name` (string)
- `emoji` (string)
- `sortOrder` (number)

#### Brand (Бренд)
- `id` (UUID, PK)
- `shopId` (UUID, FK → Shop.id)
- `categoryId` (UUID, FK → Category.id)
- `name` (string)
- `emojiPrefix` (string | null)

#### ProductFormat (Формат продукта)
- `id` (UUID, PK)
- `shopId` (UUID, FK → Shop.id)
- `brandId` (UUID, FK → Brand.id)
- `name` (string)
- `strengthLabel` (string | null) - крепость для жидкостей/снюса
- `unitPrice` (number)
- `isLiquid` (boolean)
- `isDevice` (boolean)
- `isSnus` (boolean)
- `isConsumable` (boolean)

#### Flavor (Вкус/Цвет)
- `id` (UUID, PK)
- `shopId` (UUID, FK → Shop.id)
- `productFormatId` (UUID, FK → ProductFormat.id)
- `name` (string)
- `barcode` (string | null)
- `isActive` (boolean)

#### StockItem (Остаток на складе)
- `id` (UUID, PK)
- `shopId` (UUID, FK → Shop.id)
- `flavorId` (UUID, FK → Flavor.id)
- `quantity` (number)
- `costPrice` (number) - себестоимость
- `reservedQuantity` (number) - зарезервированное количество

#### Sale (Продажа)
- `id` (UUID, PK)
- `shopId` (UUID, FK → Shop.id)
- `datetime` (Date)
- `status` ('completed' | 'reserved' | 'deleted')
- `reservationExpiry` (Date | null)
- `totalAmount` (number)

#### SaleItem (Элемент продажи)
- `id` (UUID, PK)
- `saleId` (UUID, FK → Sale.id)
- `flavorId` (UUID, FK → Flavor.id)
- `productNameSnapshot` (string)
- `flavorNameSnapshot` (string)
- `unitPrice` (number)
- `costPriceSnapshot` (number) - себестоимость на момент продажи
- `quantity` (number)
- `lineTotal` (number)

#### PostFormat (Формат поста)
- `id` (UUID, PK)
- `name` (string)
- `template` (string) - шаблон с переменными {content}
- `isActive` (boolean)
- `createdAt`, `updatedAt` (Date)

#### PostFormatSuggestion (Предложение формата)
- `id` (UUID, PK)
- `userId` (UUID, FK → User.id)
- `text` (string)
- `status` ('pending' | 'approved' | 'rejected')
- `createdAt`, `updatedAt` (Date)

### Связи между сущностями
- **Shop** → **User** (ownerId)
- **Category** → **Shop** (shopId)
- **Brand** → **Category** (categoryId), **Shop** (shopId)
- **ProductFormat** → **Brand** (brandId), **Shop** (shopId)
- **Flavor** → **ProductFormat** (productFormatId), **Shop** (shopId)
- **StockItem** → **Flavor** (flavorId), **Shop** (shopId)
- **Sale** → **Shop** (shopId)
- **SaleItem** → **Sale** (saleId), **Flavor** (flavorId)

## Авторизация и безопасность

### Методы авторизации

#### 1. Telegram Mini App
- Использование `initData` из `window.Telegram.WebApp.initData`
- Валидация подписи через `TELEGRAM_BOT_TOKEN`
- Создание/поиск пользователя по `telegramId`
- Создание сессии в HttpOnly cookie

#### 2. Access Key (для dev и прямого входа)
- Уникальный ключ для каждого пользователя
- Хранение в БД (`User.accessKey`)
- Вход через форму на странице `/login`
- Создание сессии в HttpOnly cookie

### Сессии
- Хранение в HttpOnly cookie
- Формат: `{ userId, shopId, telegramId }`
- Валидация при каждом запросе через middleware

### Роли и доступ

#### Seller (Продавец)
- Доступ ко всем функциям продавца
- Управление складом, продажами, долгами, отчетами
- Генерация постов

#### Client (Клиент)
- Ограниченный доступ (пока не реализован)
- Редирект на страницу `/client`

#### Admin (Администратор)
- Полный доступ ко всем функциям
- Доступ к админ-панели (`/admin`)
- Управление пользователями, форматами постов, категориями
- Модерация предложений форматов

### Подписки

#### Trial (Пробный период)
- По умолчанию 7 дней при регистрации
- Полный доступ к функционалу
- Автоматическое истечение через 7 дней

#### Active (Активная подписка)
- Платная подписка
- Устанавливается администратором
- Имеет дату окончания (`subscriptionEndsAt`)

#### Expired (Истекшая подписка)
- Нет доступа к функционалу
- Редирект на страницу `/subscription-expired`
- Ссылка на поддержку для продления

## Функционал

### 1. Склад (Inventory)

#### Прием товара
- Сканирование штрихкода (камера или внешний сканер)
- Ручной ввод штрихкода
- Выбор товара из каталога
- Создание нового товара при отсутствии
- Указание количества и себестоимости
- Предпросмотр товара в посте (будущее)

#### Просмотр склада
- Древовидная структура: Категория → Бренд → Формат → Вкус
- Отображение количества на складе
- Отображение себестоимости
- Фильтры:
  - Только товары в наличии
  - Только товары без штрихкода
- Сортировка по категориям, затем по брендам, затем по форматам

#### Редактирование товара
- Изменение названия вкуса
- Изменение штрихкода
- Изменение цены (unitPrice)
- Изменение себестоимости (costPrice)
- Активация/деактивация товара

#### Создание товара
- Выбор категории
- Выбор или создание бренда
- Для жидкостей: крепость (mg) при создании нового бренда
- Для устройств: цвет (flavor)
- Для снюса: крепость (mg) всегда
- Для расходников: омы (например, 0.4, 0.6, 1)
- Указание цены и себестоимости

### 2. Продажи (Sales)

#### Создание продажи
- Выбор товаров из каталога (только товары в наличии)
- Добавление в корзину с указанием количества
- Проверка доступного количества перед добавлением
- Создание продажи с указанием даты
- Возможность создания резерва (с датой истечения)

#### Просмотр продаж
- Список всех продаж
- Фильтрация по статусу (завершена, резерв, удалена)
- Фильтрация по дате
- Отображение суммы продажи

#### Редактирование продажи
- Изменение даты продажи
- Изменение статуса
- Изменение даты истечения резерва
- Удаление продажи (мягкое удаление)

### 3. Долги (Debts)

#### Создание долга
- Привязка к продаже
- Указание суммы долга
- Указание должника (имя, контакты)

#### Просмотр долгов
- Список всех долгов
- Отображение суммы долга
- Отображение даты создания
- Статус (оплачен/не оплачен)

#### Оплата долга
- Принятие оплаты по долгу
- Частичная оплата (если поддерживается)
- История операций по долгу

### 4. Отчеты (Reports)

#### Дневные отчеты
- Группировка продаж по дням
- Выручка за день
- Себестоимость за день
- Чистая прибыль за день
- Количество продаж за день

#### Фильтрация
- За последние 7/30/90 дней
- За все время
- Произвольный период (от-до)

#### Детализация
- Просмотр всех продаж за день
- Детали каждой продажи
- Прибыль по каждой продаже

### 5. Генератор постов (Post Generator)

#### Генерация поста
- Автоматическое формирование поста из товаров на складе
- Выбор формата поста (из списка активных форматов)
- Предпросмотр поста перед копированием
- Копирование поста в буфер обмена

#### Форматы постов
- Выбор формата из списка
- Предложение нового формата (отправка на модерацию)
- Использование переменных в шаблоне ({content})

### 6. Профиль (Profile)

#### Настройки магазина
- Название магазина
- Адрес магазина
- Валюта (BYN, USD, RUB)
- Тема интерфейса (светлая/темная)
- Ссылка на поддержку (Telegram username)

#### Информация о пользователе
- Роль
- Статус подписки
- Дата окончания пробного периода/подписки
- Referral код
- Access Key (для входа)

### 7. Админ-панель (Admin Panel)

#### Управление пользователями
- Список всех пользователей
- Поиск пользователей
- Изменение роли
- Изменение статуса подписки
- Установка даты окончания подписки
- Блокировка/разблокировка пользователя
- Просмотр рефералов пользователя

#### Управление форматами постов
- Список всех форматов
- Создание формата
- Редактирование формата
- Удаление формата
- Активация/деактивация формата

#### Модерация предложений
- Список предложенных форматов
- Одобрение предложения (создание формата)
- Отклонение предложения
- Фильтрация по статусу

#### Управление категориями
- Список всех категорий
- Создание категории
- Редактирование категории
- Удаление категории (с проверкой наличия товаров)

#### Настройки
- Telegram username для поддержки
- Другие глобальные настройки (будущее)

#### Статистика
- Общее количество пользователей
- Количество активных подписок
- Количество пользователей на пробном периоде
- Новые пользователи за сегодня

## Развертывание

### Переменные окружения
```env
# Database
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_password
DB_NAME=telegram_seller

# Telegram
TELEGRAM_BOT_TOKEN=your_bot_token

# Next.js
NODE_ENV=development|production
NEXT_PUBLIC_APP_URL=https://your-domain.com
```

### Требования к серверу
- **Минимум:** 1 CPU, 512MB RAM, 10GB SSD
- **Рекомендуется:** 2 CPU, 1GB RAM, 20GB SSD
- **PostgreSQL:** Версия 12+ с поддержкой UUID
- **Node.js:** Версия 18+

### Процесс развертывания
1. Установка зависимостей: `npm install`
2. Настройка переменных окружения
3. Создание БД: `psql -U postgres -c "CREATE DATABASE telegram_seller;"`
4. Запуск seed скрипта: `npm run db:seed`
5. Сборка проекта: `npm run build`
6. Запуск: `npm start`
