# Руководство по созданию администратора

## Ключ для `npm run db:seed`

В **`.env`** задайте **`DEV_SEED_ACCESS_KEY`** — им будет пользоваться тестовый аккаунт `dev-user-1`. См. `.env.example`.

⚠️ **ВАЖНО:** Не коммитьте реальные ключи в git.

## Создание администратора

Для создания администратора используйте команду:

```bash
npm run make-admin <telegramId или accessKey>
```

### Примеры использования:

1. **По Telegram ID:**
   ```bash
   npm run make-admin dev-user-1
   ```

2. **По Access Key:**
   ```bash
   npm run make-admin KEY-xxxxxxxx
   ```

### Что делает команда:

- Назначает пользователю роль `admin`
- Устанавливает статус подписки `active`
- Устанавливает срок действия подписки на 10 лет вперед

### Проверка существующих пользователей:

Если пользователь не найден, команда покажет список всех доступных пользователей с их `telegramId`, `accessKey` и текущей ролью.

## Доступ к админ-панели

После назначения администратором:

1. Войдите в приложение через веб-интерфейс или Telegram
2. В нижней навигации появится кнопка "Админ" (только для администраторов)
3. Или перейдите напрямую по адресу `/admin`

## Функции админ-панели

- **Обзор** (`/admin`) - общая статистика пользователей и подписок
- **Статистика бота** (`/admin/stats`) - детальная статистика использования бота по пользователям
- **Пользователи** (`/admin/users`) - управление пользователями, подписками и ролями
- **Форматы постов** (`/admin/formats`) - управление форматами постов
- **Предложения** (`/admin/suggestions`) - просмотр предложений пользователей
- **Сервер** (`/admin/server`) - управление сервером и режимом обслуживания
- **Настройки** (`/admin/settings`) - настройки системы

## Управление сервером

### Режим обслуживания

Режим обслуживания позволяет временно заблокировать доступ обычных пользователей к приложению, сохраняя доступ для администраторов. Это полезно при обновлении системы.

**Как использовать:**

1. Перейдите на страницу `/admin/server`
2. Включите переключатель "Режим обслуживания"
3. При необходимости укажите сообщение для пользователей
4. После завершения работ выключите режим обслуживания

**Что происходит при включении:**

- Обычные пользователи получают сообщение об обслуживании (503 ошибка)
- Администраторы могут продолжать работать с системой
- API endpoints для админов продолжают работать

### Информация о сервере

На странице `/admin/server` отображается:

- Версия приложения
- Время работы сервера (uptime)
- Статус подключения к базе данных
- Количество активных пользователей за последний час
- Текущий режим обслуживания

## Статистика использования бота

Система автоматически собирает статистику использования бота и сайта:

### Собираемые метрики:

- **Дни использования** - количество уникальных дней, когда пользователь использовал бота
- **Частота запусков** - общее количество сессий
- **Использование функций:**
  - Просмотры склада
  - Созданные продажи
  - Сгенерированные посты
  - Управление долгами
  - Просмотры отчётов

### Архитектура статистики:

- **Хранение на клиенте** - данные хранятся в `localStorage` для минимизации нагрузки
- **Периодическая синхронизация** - синхронизация с сервером каждые 5 минут
- **Умная статистика** - показывает только полезные метрики:
  - Активные пользователи (за последние 7 дней)
  - Вовлеченность (пользователи, использующие 3+ функции)
  - Топ активных пользователей
  - Распределение по подпискам
  - Средние показатели использования

### Доступ к статистике:

Перейдите в админ-панель → **Статистика бота** (`/admin/stats`) для просмотра детальной аналитики с графиками и таблицами.
