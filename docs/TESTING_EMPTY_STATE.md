# Тестирование пустого состояния приложения

Это руководство поможет вам протестировать, как приложение выглядит и работает для нового пользователя без данных в базе данных.

## Способы тестирования

### Способ 1: Очистка существующей БД (рекомендуется)

Самый простой способ - очистить все данные из текущей базы данных, сохранив структуру таблиц:

```bash
npm run db:clear
```

**Что делает скрипт:**
- Удаляет все данные из всех таблиц
- Сохраняет структуру таблиц (они остаются, но пустые)
- Дает 5 секунд на отмену операции

**После очистки:**
1. Запустите приложение: `npm run dev`
2. Откройте `/login` для входа (создастся новый пользователь)
3. Протестируйте все разделы приложения с пустыми данными

**Для восстановления тестовых данных:**
```bash
npm run db:seed
```

### Способ 2: Создание новой пустой БД

Создайте новую базу данных для тестирования:

```bash
# 1. Создайте новую БД
psql -U postgres -c "CREATE DATABASE telegram_seller_empty;"

# 2. Обновите .env файл
# Измените DB_NAME на telegram_seller_empty

# 3. Запустите приложение
npm run dev
# При первом запросе TypeORM создаст все таблицы автоматически

# 4. Войдите через /login
```

**Для возврата к основной БД:**
Просто верните `DB_NAME` в `.env` обратно к `telegram_seller`.

### Способ 3: Использование SQL для очистки

Если хотите очистить БД вручную через SQL:

```sql
-- Подключитесь к БД
psql -U postgres -d telegram_seller

-- Удалите данные из всех таблиц (в правильном порядке)
TRUNCATE TABLE debt_operations CASCADE;
TRUNCATE TABLE debts CASCADE;
TRUNCATE TABLE sale_items CASCADE;
TRUNCATE TABLE sales CASCADE;
TRUNCATE TABLE stock_items CASCADE;
TRUNCATE TABLE flavors CASCADE;
TRUNCATE TABLE product_formats CASCADE;
TRUNCATE TABLE brands CASCADE;
TRUNCATE TABLE categories CASCADE;
TRUNCATE TABLE post_format_suggestions CASCADE;
TRUNCATE TABLE post_formats CASCADE;
TRUNCATE TABLE user_shops CASCADE;
TRUNCATE TABLE shops CASCADE;
TRUNCATE TABLE users CASCADE;
```

## Восстановление данных

После тестирования вы можете восстановить тестовые данные:

```bash
npm run db:seed
```

Или вернуться к основной БД, если использовали отдельную базу данных.

## Важные замечания

⚠️ **ВНИМАНИЕ:** 
- Скрипт `db:clear` удаляет ВСЕ данные без возможности восстановления
- Рекомендуется делать резервную копию БД перед очисткой
- В production окружении этот скрипт не должен быть доступен

💡 **Совет:**
- Используйте отдельную БД для тестирования пустого состояния
- Или делайте резервную копию перед очисткой: `pg_dump -U postgres telegram_seller > backup.sql`
- Для восстановления: `psql -U postgres telegram_seller < backup.sql`
