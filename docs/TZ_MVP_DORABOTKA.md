#### 3.1.3. Расширение `StockItemEntity`
**Файл**: `src/lib/db/entities/StockItem.ts`

Добавить поле:
- `reservedQuantity: number` - количество забронированных единиц. По умолчанию `0`.

**Логика**: 
- `quantity` - общее количество на складе
- `reservedQuantity` - количество в брони
- Доступное количество = `quantity - reservedQuantity`
- При создании брони: `reservedQuantity` увеличивается, `quantity` не меняется
- При подтверждении брони: `reservedQuantity` уменьшается, `quantity` уменьшается
- При отмене брони: `reservedQuantity` уменьшается, `quantity` не меняется

```typescript
export interface StockItem {
  // ... существующие поля
  reservedQuantity: number; // default: 0
}
```

### 3.2. API Endpoints

#### 3.2.1. Редактирование продажи
**Файл**: `src/app/api/sales/[id]/route.ts` (новый)

**Методы**:
- `GET /api/sales/[id]` - получение детальной информации о продаже
- `PATCH /api/sales/[id]` - редактирование продажи
- `DELETE /api/sales/[id]` - удаление продажи с возвратом товара на склад

**PATCH /api/sales/[id]**
- Поля для редактирования:
  - `datetime: Date` - дата и время продажи
  - `paymentType: 'cash' | 'card' | 'debt'`
  - `discountValue: number`
  - `discountType: 'absolute' | 'percent'`
  - `comment: string | null`
  - `customerName: string | null` (если `paymentType === 'debt'`)
  - `items: Array<{ flavorId, quantity, unitPrice }>` - изменение состава продажи
  - `isReservation: boolean` - переключение статуса брони

**Логика редактирования**:
1. Если изменены `items`:
   - Вычислить разницу в количестве для каждого товара
   - Вернуть излишки на склад (увеличить `quantity` в `StockItem`)
   - Уменьшить недостающие количества (проверить доступность)
   - Обновить `SaleItem` записи
2. Если изменена дата (`datetime`):
   - Обновить только поле `datetime` в `Sale`
   - Обновить `updatedAt`
3. Если изменен `isReservation`:
   - При переходе из брони в продажу: уменьшить `reservedQuantity`, уменьшить `quantity`
   - При переходе из продажи в бронь: увеличить `reservedQuantity`, вернуть `quantity`
4. Установить `status: 'edited'` после редактирования
5. Пересчитать `totalAmount`, `finalAmount`, `totalCost` на основе новых данных

**DELETE /api/sales/[id]**
- Установить `status: 'deleted'` в `Sale`
- Вернуть все товары на склад:
  - Если `isReservation === true`: уменьшить `reservedQuantity`
  - Если `isReservation === false`: увеличить `quantity`
- Если продажа была в долг (`paymentType === 'debt'`):
  - Уменьшить `totalDebt` в `DebtEntity`
  - Создать запись в `DebtOperationEntity` с комментарием об отмене

**Валидация**:
- Проверка прав доступа (только продавец/владелец магазина)
- Проверка доступности товаров при изменении количества
- Проверка корректности даты (не в будущем, не слишком давно)

  - `showReserved: boolean` - показывать ли забронированные товары отдельно

- `strengthFilter === "all"` или отсутствует - без фильтрации

**Ответ API**:

## 5. Технические детали

### 5.1. Валидация валюты
Список поддерживаемых валют хранить в константе:
```typescript
export const SUPPORTED_CURRENCIES = ['BYN', 'RUB', 'USD', 'EUR'] as const;
export type Currency = typeof SUPPORTED_CURRENCIES[number];
```

### 5.2. Форматирование даты
Использовать `date-fns` с локализацией `ru` для всех дат в UI.

### 5.3. Состояние фильтров
Сохранять состояние фильтров в `useState` и синхронизировать с URL query params через `useSearchParams` (Next.js).

### 5.4. Обработка транзакций
Все операции изменения продаж должны выполняться в транзакциях TypeORM для обеспечения целостности данных.

### 5.5. Кэширование
Использовать React Query для кэширования данных:
- Инвалидировать кэш после редактирования/удаления продажи
- Инвалидировать кэш инвентаря после изменения брони

## 8. Примечания

- Все тексты в UI должны быть на русском языке
- Все суммы должны отображаться с символом валюты
- Даты должны отображаться в формате, понятном пользователю (например, "3 марта 2026, 14:30")
- При работе с бронями нужно учитывать возможные конфликты (два продавца одновременно бронируют один товар)
- При удалении продажи нужно проверять, не была ли она уже отредактирована (чтобы не потерять историю)
