## 1. Авторизация и безопасность

### Методы авторизации

#### 1.1. Авторизация через Telegram Mini App
- **Путь:** `/api/auth/telegram`
- **Метод:** POST
- **Описание:** Авторизация через `initData` из `window.Telegram.WebApp.initData`
- **Функционал:**
  - Валидация подписи через `TELEGRAM_BOT_TOKEN`
  - Создание нового пользователя при первом входе
  - Автоматическое создание магазина для нового пользователя
  - Создание пробного периода на 7 дней
  - Генерация уникального `accessKey` для входа без Telegram
  - Генерация реферального кода
  - Создание сессии в HttpOnly cookie

#### 1.2. Dev-авторизация (только для разработки)
- **Путь:** `/api/auth/dev`
- **Метод:** POST
- **Описание:** Авторизация без Telegram для локальной разработки
- **Функционал:**
  - Создание тестового пользователя с `telegramId: 'dev-user-1'`
  - Автоматическое создание магазина
  - Создание сессии в HttpOnly cookie
  - Доступна только при `NODE_ENV=development`

#### 1.3. Авторизация по ключу доступа
- **Путь:** `/api/auth/key`
- **Метод:** POST
- **Описание:** Вход по уникальному ключу доступа (`accessKey`)
- **Функционал:**
  - Поиск пользователя по `accessKey`
  - Создание сессии в HttpOnly cookie
  - Доступ к приложению без Telegram

### Безопасность

- **Подпись сессий:** Все сессии подписываются секретным ключом (`SESSION_SECRET`)
- **HttpOnly cookies:** Сессии хранятся в защищённых cookies
- **Изоляция данных:** Каждый пользователь видит только данные своего магазина
- **Проверка прав доступа:** Все API endpoints проверяют сессию и права доступа
- **Защита админ-панели:** Доступ только для пользователей с ролью `admin`

### Роли пользователей

- **Seller (Продавец):** Полный доступ ко всем функциям продавца
- **Client (Клиент):** Ограниченный доступ (пока не реализован полностью)
- **Admin (Администратор):** Полный доступ + админ-панель

### Подписки

- **Trial (Пробный период):** 7 дней при регистрации, полный доступ
- **Active (Активная):** Платная подписка, устанавливается администратором
- **Expired (Истекшая):** Нет доступа к функционалу, редирект на страницу продления

## 2. Дашборд

**Путь:** `/dashboard` или `/`

### Основные функции

#### 2.1. Информация о магазине
- Отображение названия магазина
- Адрес магазина (если указан)
- Валюта магазина

#### 2.2. Статистика за сегодня
- **Выручка:** Общая сумма продаж за сегодня
- **Долги:** Общая сумма всех долгов
- **Склад:** Общее количество товаров на складе
- **Резерв:** Количество зарезервированных товаров

#### 2.3. Сводка за сегодня
- Количество продаж
- Наличные (cash)
- Карта (card)
- В долг (debt)
- Резервы (количество и сумма)
- Итоговая выручка

#### 2.4. Быстрые действия
- **Новая продажа:** Переход на страницу создания продажи
- **Принять товар:** Модальное окно для приёма товара на склад
- **Склад:** Переход на страницу склада
- **Пост в чат:** Переход на генератор постов
- **Профиль:** Переход в профиль

#### 2.5. Топ долгов
- Отображение 3 должников с наибольшей суммой долга
- Имя должника
- Дата последнего обновления
- Сумма долга
- Переход к полному списку долгов

#### 2.6. Топ резервов
- Отображение 3 активных резервов
- Имя клиента
- Время до истечения резерва
- Сумма резерва
- Цветовая индикация срока действия (зелёный/жёлтый/красный)

#### 2.7. Последние дни
- Отображение продаж за последние 3 дня
- Дата
- Количество продаж
- Описание последней продажи
- Выручка за день
- Переход к полным отчётам

## 3. Склад (Inventory)

**Путь:** `/inventory`

### Основные функции

#### 3.1. Просмотр склада
- **Древовидная структура:**
  - Категория → Бренд → Формат → Вкус/Цвет
- **Отображение информации:**
  - Количество на складе
  - Зарезервированное количество
  - Себестоимость
  - Цена продажи
  - Штрихкод (если есть)
- **Раскрытие/сворачивание:** Каждый уровень можно раскрыть/свернуть

#### 3.2. Поиск товаров
- Поиск по названию товара, бренда, категории
- Поиск по штрихкоду
- Мгновенный поиск при вводе

#### 3.3. Фильтры
- **Только товары в наличии:** Показать только товары с количеством > 0
- **Только без штрихкода:** Показать товары без штрихкода
- **Только резервы:** Показать только зарезервированные товары
- **По категории:** Фильтр по выбранной категории
- **По бренду:** Фильтр по выбранному бренду
- **По крепости:** Фильтр по крепости (для жидкостей/снюса)
- **По цвету:** Фильтр по цвету (для устройств)
- **По цене:** Минимальная и максимальная цена

#### 3.4. Приём товара на склад
- **Сканирование штрихкода:**
  - Использование камеры устройства
  - Поддержка внешних сканеров штрихкодов
- **Ручной ввод штрихкода**
- **Выбор товара из каталога:**
  - Поиск по названию
  - Выбор категории, бренда, формата, вкуса
- **Создание нового товара:**
  - Если товар не найден, можно создать новый
  - Выбор категории
  - Создание или выбор бренда
  - Указание формата продукта
  - Указание вкуса/цвета
- **Указание параметров:**
  - Количество
  - Себестоимость
  - Штрихкод (если не был отсканирован)

#### 3.5. Редактирование товара
- **Изменение названия вкуса/цвета**
- **Изменение штрихкода**
- **Изменение цены продажи** (`unitPrice`)
- **Изменение себестоимости** (`costPrice`)
- **Активация/деактивация товара**

#### 3.6. Редактирование бренда и формата
- Изменение названия бренда
- Изменение названия формата
- Изменение цены формата (применяется ко всем вкусам)

#### 3.7. Управление остатками
- **Увеличение количества:** Быстрое добавление товара на склад
- **Уменьшение количества:** Списание товара со склада
- **Прямое редактирование:** Установка точного количества

#### 3.8. Продажа резерва
- Конвертация резерва в продажу
- Выбор способа оплаты
- Указание клиента

#### 3.9. Управление категориями
- Просмотр всех категорий
- Создание новой категории
- Редактирование категории (название, эмодзи)
- Удаление категории (с проверкой наличия товаров)
- Изменение порядка сортировки категорий

#### 3.10. Управление брендами
- Просмотр всех брендов
- Создание нового бренда
- Редактирование бренда (название, эмодзи-префикс)
- Удаление бренда (с проверкой наличия товаров)
- Изменение порядка сортировки брендов в категории

## 4. Продажи (Sales)

**Путь:** `/sales`

### Основные функции

#### 4.1. Создание продажи
- **Выбор товаров:**
  - Поиск товаров по названию или штрихкоду
  - Показываются только товары в наличии (не в резервах)
  - Добавление товара в корзину
- **Корзина:**
  - Просмотр выбранных товаров
  - Изменение количества (с проверкой доступности)
  - Удаление товара из корзины
  - Автоматический расчёт суммы
- **Параметры продажи:**
  - Способ оплаты: наличные, карта, в долг
  - Скидка: процентная или фиксированная
  - Имя клиента (опционально)
  - Комментарий (опционально)
  - Дата и время продажи
- **Создание резерва:**
  - Возможность создать резерв вместо продажи
  - Указание срока действия резерва
  - Имя клиента для резерва

#### 4.2. Просмотр продаж
- **Список всех продаж:**
  - Отображение в хронологическом порядке
  - Информация о каждой продаже:
    - Дата и время
    - Список товаров
    - Способ оплаты
    - Сумма
    - Статус (завершена, резерв, удалена)
- **Фильтрация:**
  - По статусу (все, завершённые, резервы, удалённые)
  - По дате
  - По способу оплаты
- **Поиск:**
  - По названию товара
  - По имени клиента

#### 4.3. Редактирование продажи
- **Изменение даты продажи**
- **Изменение способа оплаты**
- **Изменение скидки**
- **Изменение имени клиента**
- **Изменение комментария**
- **Изменение статуса:**
  - Завершена → Резерв
  - Резерв → Завершена
  - Удаление продажи (мягкое удаление)
- **Изменение даты истечения резерва**
- **Редактирование товаров в продаже:**
  - Добавление товара
  - Удаление товара
  - Изменение количества
  - Замена товара

#### 4.4. Удаление продажи
- Мягкое удаление (статус меняется на `deleted`)
- Восстановление остатков на складе
- Удаление связанных долгов

## 5. Долги (Debts)

**Путь:** `/debts` или вкладка "Долги" в профиле

### Основные функции

#### 5.1. Просмотр долгов
- **Список всех должников:**
  - Имя должника
  - Общая сумма долга
  - Дата последнего обновления
  - История операций по долгу
- **Общая статистика:**
  - Общая сумма всех долгов
  - Количество должников

#### 5.2. Детали долга
- Просмотр истории операций:
  - Дата и время операции
  - Сумма (положительная - долг, отрицательная - оплата)
  - Комментарий
- Связь с продажами (если долг создан из продажи)

#### 5.3. Принятие оплаты
- **Запись оплаты:**
  - Указание суммы оплаты
  - Комментарий (опционально)
  - Автоматическое обновление суммы долга
- **Частичная оплата:** Поддержка частичных платежей
- **Полная оплата:** Автоматическое закрытие долга при полной оплате

#### 5.4. Создание долга
- Долг создаётся автоматически при продаже с оплатой "в долг"
- Возможность создать долг вручную (через продажу)

#### 6.1. Дневные отчёты

  - Дата

  - Выручка за день
  - Себестоимость за день

  - Скидки

  - Неделя (последние 7 дней)

    - Скидка
    - Сумма

#### 6.4. Графики и визуализация
- **График выручки:** Линейный график выручки по дням
- **График прибыли:** Линейный график прибыли по дням

- **Распределение по способам оплаты:** Круговая диаграмма
- **Сравнение периодов:** Сравнение с предыдущим периодом

#### 6.5. Экспорт данных
- Экспорт отчётов (планируется)

## 8. Профиль и настройки

- Полный список всех долгов

- Принятие оплаты

  - Поделиться ссылкой через Telegram

## 9. Админ-панель

**Путь:** `/admin` (доступна только администраторам)

### Основные разделы

#### 9.1. Обзор (Dashboard)
- **Общая статистика:**
  - Общее количество пользователей
  - Активные подписки
  - Пользователи на пробном периоде
  - Новые пользователи за сегодня
  - Пользователи с истекшей подпиской

#### 9.2. Статистика бота (`/admin/stats`)
- **Метрики использования:**
  - Дни использования (уникальные дни активности)
  - Частота запусков (общее количество сессий)
  - Использование функций:
    - Просмотры склада
    - Созданные продажи
    - Сгенерированные посты
    - Управление долгами
    - Просмотры отчётов
- **Аналитика:**
  - Активные пользователи (за последние 7 дней)
  - Вовлечённость (пользователи, использующие 3+ функции)
  - Топ активных пользователей
  - Распределение по подпискам
  - Средние показатели использования
- **Графики и визуализация:**
  - Графики активности пользователей
  - Распределение по функциям

#### 9.3. Управление пользователями (`/admin/users`)
- **Список пользователей:**
  - Поиск по имени, username, telegramId, accessKey
  - Фильтрация по роли, статусу подписки
  - Сортировка
- **Редактирование пользователя:**
  - Изменение роли (Seller/Client/Admin)
  - Изменение статуса подписки (Trial/Active/Expired)
  - Установка даты окончания подписки
  - Блокировка/разблокировка пользователя
  - Просмотр информации о пользователе
- **Рефералы пользователя:**
  - Просмотр всех рефералов пользователя
  - Статистика по рефералам

#### 9.4. Управление форматами постов (`/admin/formats`)
- **Список форматов:**
  - Все форматы постов
  - Статус (активен/неактивен)
  - Дата создания и обновления
- **Создание формата:**
  - Название формата
  - Шаблон с переменной `{content}`
  - Активация формата
- **Редактирование формата:**
  - Изменение названия
  - Изменение шаблона
  - Активация/деактивация
- **Удаление формата**

#### 9.5. Модерация предложений (`/admin/suggestions`)
- **Список предложений:**
  - Все предложенные форматы

- Другие административные настройки

## 10. Telegram бот

### Команды бота

#### 10.1. `/start`
- **Описание:** Приветствие и онбординг
- **Функционал:**
  - Приветственное сообщение с описанием бота
  - Выбор роли для новых пользователей (Продавец/Клиент)
  - Поддержка реферальных ссылок
  - Для существующих пользователей - информация о профиле
  - Кнопка открытия веб-приложения

#### 10.2. `/key`
- **Описание:** Получение ключа доступа
- **Функционал:**
  - Отображение уникального `accessKey` пользователя
  - Инструкция по использованию ключа
  - Кнопка копирования ключа

#### 10.3. `/me`
- **Описание:** Информация о профиле
- **Функционал:**
  - Карточка пользователя:
    - ID пользователя
    - Текущая роль
    - Статус подписки с количеством дней
    - Статистика (рефералы, созданные форматы)
  - Inline-кнопки для навигации:
    - Управление подпиской
    - Переключение роли (Продавец ↔ Клиент)
    - Реферальная программа
    - Мой ключ доступа
    - Назад

#### 10.4. `/subscribe`
- **Описание:** Управление подпиской
- **Функционал:**
  - Описание тарифного плана PRO
  - Список возможностей
  - Информация о текущей подписке
  - Кнопка покупки через Telegram Stars

#### 10.5. `/referrals`
- **Описание:** Реферальная программа
- **Функционал:**
  - Статистика:
    - Переходы по ссылке
    - Регистрации
    - Оплатившие подписку
  - Кнопка "Поделиться ссылкой" (switch_inline_query)
  - Кнопка "Копировать ссылку"

#### 10.6. `/role`
- **Описание:** Переключение роли
- **Функционал:**
  - Переключение между ролями Продавец и Клиент
  - Подтверждение смены роли

### Callback handlers

- **`role_seller` / `role_client`:** Выбор роли при регистрации
- **`copy_post`:** Копирование поста (планируется)
- **Навигационные callback:** Переходы между разделами бота

### Webhook

- **Путь:** `/api/telegram/webhook`
- **Метод:** POST
- **Функционал:**
  - Приём обновлений от Telegram
  - Обработка команд и callback-запросов
  - Установка/удаление webhook
  - Проверка статуса webhook
