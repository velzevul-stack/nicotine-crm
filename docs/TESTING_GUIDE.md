## 1. Подготовка окружения

### 1.1. Установка PostgreSQL

**Windows:**
1. Скачайте PostgreSQL с официального сайта: https://www.postgresql.org/download/windows/
2. Установите PostgreSQL (запомните пароль для пользователя `postgres`)
3. Убедитесь, что служба PostgreSQL запущена

**Проверка установки:**
```bash
psql --version
```

### 1.2. Создание базы данных

Откройте командную строку или PowerShell и выполните:

```bash
# Подключение к PostgreSQL (введите пароль при запросе)
psql -U postgres

# Создание базы данных
CREATE DATABASE telegram_seller;

# Выход из psql
\q
```

Или одной командой:
```bash
psql -U postgres -c "CREATE DATABASE telegram_seller;"
```

### 1.3. Установка Node.js

1. Скачайте Node.js версии 18 или выше: https://nodejs.org/
2. Установите Node.js
3. Проверьте установку:
```bash
node --version
npm --version
```

### 1.4. Настройка переменных окружения

1. Скопируйте файл `.env.example` в `.env`:
```bash
# Windows PowerShell
Copy-Item .env.example .env

# Windows CMD
copy .env.example .env
```

2. Откройте файл `.env` и заполните следующие переменные:

```env
# База данных PostgreSQL
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=ваш_пароль_postgres
DB_NAME=telegram_seller

# Режим разработки
NODE_ENV=development

# Telegram Bot Token (получите у @BotFather в Telegram)
TELEGRAM_BOT_TOKEN=ваш_токен_бота

```

**Важно:** 
- Замените `ваш_пароль_postgres` на реальный пароль от PostgreSQL
- Для локального тестирования `TELEGRAM_BOT_TOKEN` можно оставить пустым (но тогда не будет работать авторизация через Telegram)

## 2. Развёртывание приложения локально

### 2.1. Установка зависимостей

Откройте терминал в корне проекта и выполните:

```bash
npm install
```

Это установит все необходимые зависимости проекта.

### 2.2. Запуск приложения в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: **http://localhost:3000**

**Что происходит при первом запуске:**
- TypeORM автоматически создаст все таблицы в базе данных (synchronize)
- Приложение будет готово к работе

### 2.3. Наполнение тестовыми данными

В новом терминале выполните:

```bash
npm run db:seed
```

Эта команда создаст:
- Тестового пользователя
- Магазин
- Категории товаров
- Бренды
- Продукты и вкусы
- Остатки на складе
- Примеры продаж и долгов

**Результат:** В консоли вы увидите сообщения о создании данных.

### 2.4. Проверка работы приложения

1. Откройте браузер и перейдите на http://localhost:3000
2. Вы должны увидеть страницу входа или автоматический вход (если используется Telegram Mini App)

## 3. Настройка Telegram бота

### 3.1. Создание бота через BotFather

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям:
   - Введите имя бота (например: "My Seller Bot")
   - Введите username бота (должен заканчиваться на `bot`, например: `my_seller_bot`)
4. BotFather выдаст вам токен вида: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`
5. Скопируйте токен и добавьте в файл `.env`:
   ```env
   TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
   ```

### 3.2. Настройка Webhook для локальной разработки

Для локального тестирования бота вам понадобится туннель (например, ngrok).

#### Вариант 1: Использование ngrok

1. Скачайте ngrok: https://ngrok.com/download
2. Запустите ngrok:
   ```bash
   ngrok http 3000
   ```
3. Скопируйте HTTPS URL (например: `https://abc123.ngrok.io`)
4. Установите webhook:
   ```bash
   # В браузере откройте:
   https://abc123.ngrok.io/api/telegram/webhook?setWebhook=true
   ```
   Или через curl:
   ```bash
   curl "https://abc123.ngrok.io/api/telegram/webhook?setWebhook=true"
   ```

5. Обновите `.env` (опционально):
   ```env
   TELEGRAM_WEBHOOK_URL=https://abc123.ngrok.io/api/telegram/webhook
   ```

#### Вариант 2: Использование локального сервера (без webhook)

Для простого тестирования команд бота можно использовать polling вместо webhook. Но для production нужен webhook.

**Важно:** При каждом перезапуске ngrok URL меняется, нужно обновлять webhook!

### 3.3. Проверка работы бота

1. Найдите вашего бота в Telegram по username
2. Отправьте команду `/start`
3. Бот должен ответить приветствием

## 4. Тестирование авторизации

### 4.1. Dev-вход (для локального тестирования)

1. Откройте браузер: http://localhost:3000/login
2. Нажмите кнопку "Войти" или "Dev Login"
3. Должен произойти автоматический вход
4. Вы будете перенаправлены на главную страницу (дашборд)

**Ожидаемый результат:**
- Создаётся тестовый пользователь с `telegramId: 'dev-user-1'`
- Создаётся сессия в cookie
- Пользователь видит дашборд приложения

### 4.2. Вход через Telegram Mini App

1. Создайте Mini App в BotFather:
   - Откройте [@BotFather](https://t.me/BotFather)
   - Отправьте `/newapp`
   - Выберите вашего бота
   - Введите название приложения
   - Загрузите иконку (опционально)
   - Введите описание
   - Укажите URL: `https://ваш_ngrok_url.ngrok.io` (или ваш production URL)
   - Укажите короткое имя для приложения

2. Откройте бота в Telegram и нажмите на кнопку "Open App" или отправьте команду `/start` и нажмите на кнопку Mini App

3. Приложение должно открыться и автоматически авторизовать вас через `initData`

**Ожидаемый результат:**
- Автоматическая авторизация через Telegram
- Создание пользователя в базе данных (если первый раз)
- Создание магазина для пользователя
- Доступ к приложению

### 4.3. Вход через Access Key

1. Получите ключ доступа через бота:
   - Откройте бота в Telegram
   - Отправьте команду `/key`
   - Бот вернёт ваш ключ доступа

2. Откройте в браузере: http://localhost:3000/login
3. Введите полученный ключ доступа
4. Нажмите "Войти"

**Ожидаемый результат:**
- Успешный вход в систему
- Перенаправление на дашборд

### 4.4. Проверка ошибок авторизации

**Тест 1: Неверный ключ доступа**
1. Откройте `/login`
2. Введите несуществующий ключ: `invalid-key-12345`
3. Нажмите "Войти"

**Ожидаемый результат:**
- Сообщение об ошибке: "Неверный ключ доступа" или подобное
- Пользователь остаётся на странице входа

**Тест 2: Пустой ключ**
1. Откройте `/login`
2. Оставьте поле пустым
3. Нажмите "Войти"

**Ожидаемый результат:**
- Валидация формы не пропускает пустое поле
- Сообщение о необходимости ввести ключ

## 5. Тестирование основных функций

### 5.1. Дашборд (главная страница)

**Путь:** `/` или автоматически после входа

**Что проверить:**

1. **Отображение статистики:**
   - Общая сумма продаж за сегодня
   - Количество продаж за сегодня
   - Количество должников
   - Остатки на складе (низкие остатки)

2. **Быстрые действия:**
   - Кнопка "Новая продажа" → должна вести на `/sales`
   - Кнопка "Склад" → должна вести на `/inventory`
   - Кнопка "Пост в чат" → должна вести на `/post`
   - Кнопка "Профиль" → должна вести на `/profile`

3. **Последние продажи:**
   - Отображается список последних продаж
   - Можно кликнуть на продажу для просмотра деталей

4. **Активные долги:**
   - Отображается список должников
   - Показывается сумма долга

**Тестовые сценарии:**

**Сценарий 1: Просмотр дашборда с данными**
1. Войдите в систему
2. Убедитесь, что данные загружаются
3. Проверьте корректность отображения статистики

**Сценарий 2: Пустой дашборд (новый пользователь)**
1. Очистите базу данных: `npm run db:clear`
2. Войдите как новый пользователь
3. Убедитесь, что дашборд показывает пустое состояние с подсказками

### 5.2. Склад (Inventory)

**Путь:** `/inventory`

**Что проверить:**

1. **Структура склада:**
   - Дерево категорий → бренды → форматы → вкусы
   - Раскрытие/сворачивание категорий
   - Поиск по товарам

2. **Управление остатками:**
   - Просмотр остатков по каждому вкусу
   - Изменение остатков (увеличение/уменьшение)
   - Приём товара на склад

3. **Управление категориями:**
   - Создание новой категории
   - Редактирование категории
   - Удаление категории
   - Изменение порядка категорий (drag & drop)

4. **Управление брендами:**
   - Создание бренда в категории
   - Редактирование бренда
   - Удаление бренда
   - Изменение порядка брендов

5. **Управление продуктами:**
   - Создание формата продукта
   - Добавление вкусов к формату
   - Редактирование вкусов
   - Удаление вкусов

**Тестовые сценарии:**

**Сценарий 1: Просмотр склада**
1. Перейдите на `/inventory`
2. Убедитесь, что дерево категорий загружается
3. Раскройте категорию → бренд → формат
4. Проверьте отображение остатков

**Сценарий 2: Изменение остатка**
1. Найдите любой товар на складе
2. Нажмите на кнопку изменения остатка (обычно "+" или "−")
3. Введите новое значение или используйте кнопки +/- 
4. Сохраните изменения
5. Убедитесь, что остаток обновился

**Сценарий 3: Приём товара на склад**
1. На дашборде нажмите "Принять товар" (если есть такая кнопка)
2. Или перейдите в раздел склада
3. Выберите товар
4. Введите количество для приёма
5. Сохраните
6. Убедитесь, что остаток увеличился

**Сценарий 4: Создание новой категории**
1. Перейдите на `/inventory`
2. Нажмите "Добавить категорию" или "+"
3. Введите название категории (например: "Электронные сигареты")
4. Сохраните
5. Убедитесь, что категория появилась в списке

**Сценарий 5: Создание бренда**
1. Раскройте категорию
2. Нажмите "Добавить бренд"
3. Введите название бренда (например: "Vaporesso")
4. Сохраните
5. Убедитесь, что бренд появился в категории

**Сценарий 6: Создание формата продукта**
1. Раскройте категорию → бренд
2. Нажмите "Добавить формат" или "Добавить продукт"
3. Заполните форму:
   - Название формата (например: "XROS 3")
   - Описание (опционально)
4. Сохраните
5. Убедитесь, что формат появился

**Сценарий 7: Добавление вкуса**
1. Раскройте формат продукта
2. Нажмите "Добавить вкус"
3. Заполните:
   - Название вкуса (например: "Манго")
   - Начальный остаток (например: 10)
4. Сохраните
5. Убедитесь, что вкус появился с указанным остатком

**Сценарий 8: Поиск товара**
1. Используйте поле поиска на странице склада
2. Введите название товара или вкуса
3. Убедитесь, что результаты фильтруются

**Сценарий 9: Изменение порядка (если поддерживается)**
1. Перетащите категорию вверх/вниз
2. Убедитесь, что порядок сохранился после перезагрузки

### 5.3. Продажи (Sales)

**Путь:** `/sales`

**Что проверить:**

1. **Создание продажи:**
   - Добавление товаров в корзину
   - Выбор количества
   - Расчет общей суммы
   - Сохранение продажи

2. **Список продаж:**
   - Отображение всех продаж
   - Фильтрация по дате
   - Поиск по продажам
   - Просмотр деталей продажи

3. **Редактирование продажи:**
   - Изменение товаров в продаже
   - Изменение количества
   - Удаление продажи

**Тестовые сценарии:**

**Сценарий 1: Создание новой продажи**
1. Перейдите на `/sales`
2. Нажмите "Новая продажа" или "+"
3. Добавьте товар в корзину:
   - Найдите товар через поиск или выбор из списка
   - Выберите вкус
   - Укажите количество
   - Нажмите "Добавить"
4. Повторите для нескольких товаров
5. Проверьте, что общая сумма рассчитывается правильно
6. Нажмите "Оформить продажу" или "Сохранить"
7. Убедитесь, что продажа создана и остатки уменьшились

**Сценарий 2: Просмотр списка продаж**
1. Перейдите на `/sales`
2. Убедитесь, что отображается список всех продаж
3. Проверьте отображение:
   - Дата продажи
   - Список товаров
   - Общая сумма
   - Количество товаров

**Сценарий 3: Фильтрация продаж по дате**
1. Используйте фильтр даты (если есть)
2. Выберите диапазон дат
3. Убедитесь, что список обновился

**Сценарий 4: Поиск продажи**
1. Используйте поле поиска
2. Введите название товара или другую информацию
3. Убедитесь, что результаты фильтруются

**Сценарий 5: Просмотр деталей продажи**
1. Кликните на любую продажу в списке
2. Убедитесь, что открывается детальная информация:
   - Дата и время
   - Список всех товаров с количеством и ценами
   - Общая сумма

**Сценарий 6: Продажа с недостаточным остатком**
1. Попробуйте продать товар, остаток которого меньше запрашиваемого количества
2. Убедитесь, что система предупреждает об ошибке
3. Проверьте, что продажа не создаётся

**Сценарий 7: Продажа несуществующего товара**
1. Попробуйте добавить в корзину товар, которого нет на складе
2. Убедитесь, что система показывает соответствующее сообщение

### 5.4. Долги (Debts)

**Путь:** `/debts`

**Что проверить:**

1. **Список должников:**
   - Отображение всех должников
   - Сумма долга для каждого
   - История операций по долгу

2. **Создание долга:**
   - Добавление нового должника
   - Указание суммы долга
   - Описание/комментарий

3. **Приём оплаты:**
   - Частичная оплата долга
   - Полная оплата долга
   - История платежей

4. **Редактирование долга:**
   - Изменение суммы
   - Добавление комментария
   - Удаление долга

**Тестовые сценарии:**

**Сценарий 1: Создание нового долга**
1. Перейдите на `/debts`
2. Нажмите "Добавить долг" или "+"
3. Заполните форму:
   - Имя должника (например: "Иван Иванов")
   - Сумма долга (например: 5000)
   - Комментарий (опционально, например: "За товары от 01.03.2025")
4. Сохраните
5. Убедитесь, что долг появился в списке

**Сценарий 2: Просмотр списка долгов**
1. Перейдите на `/debts`
2. Убедитесь, что отображаются все должники
3. Проверьте отображение:
   - Имя должника
   - Сумма долга
   - Дата создания

**Сценарий 3: Частичная оплата долга**
1. Выберите долг из списка
2. Нажмите "Принять оплату" или "Оплатить"
3. Введите сумму частичной оплаты (меньше общей суммы долга)
4. Сохраните
5. Убедитесь, что:
   - Остаток долга уменьшился
   - В истории появилась запись об оплате

**Сценарий 4: Полная оплата долга**
1. Выберите долг
2. Нажмите "Принять оплату"
3. Введите сумму, равную остатку долга
4. Сохраните
5. Убедитесь, что:
   - Долг закрыт (статус изменён)
   - Долг больше не отображается в активных долгах (или отображается как закрытый)

**Сценарий 5: Просмотр истории операций**
1. Откройте детали долга
2. Убедитесь, что отображается история:
   - Создание долга
   - Все платежи с датами и суммами
   - Текущий остаток

**Сценарий 6: Редактирование долга**
1. Выберите долг
2. Нажмите "Редактировать" (если доступно)
3. Измените сумму или комментарий
4. Сохраните
5. Убедитесь, что изменения применились

**Сценарий 7: Удаление долга**
1. Выберите долг
2. Нажмите "Удалить" (если доступно)
3. Подтвердите удаление
4. Убедитесь, что долг удалён из списка

**Сценарий 8: Поиск должника**
1. Используйте поле поиска

- **Действие:** Отправить поддельный initData на `/api/auth/telegram`
- **Ожидаемый результат:** Ошибка 401 "Invalid initData"

- **Действие:** Попытаться установить остаток меньше нуля
- **Ожидаемый результат:** Валидация не пропускает отрицательное значение

- **Ожидаемый результат:** Предупреждение, продажа не создаётся

- **Действие:** Попытаться удалить категорию, в которой есть товары
- **Ожидаемый результат:** Предупреждение или каскадное удаление (зависит от реализации)

**Тест 4: Дублирование названия категории/бренда**
- **Действие:** Создать категорию/бренд с уже существующим названием
- **Ожидаемый результат:** Валидация или предупреждение о дублировании

### 7.3. Ошибки работы с продажами

**Тест 1: Пустая корзина**
- **Действие:** Попытаться оформить продажу с пустой корзиной
- **Ожидаемый результат:** Предупреждение, продажа не создаётся

- **Действие:** Попытаться добавить в корзину товар, который был удалён
- **Ожидаемый результат:** Ошибка или предупреждение

- **Действие:** Создать долг с отрицательной суммой
- **Ожидаемый результат:** Валидация не пропускает

- **Действие:** Попытаться принять оплату больше, чем остаток долга

**Тест 1: Потеря соединения**
- **Действие:** Отключить интернет и попытаться выполнить действие
- **Ожидаемый результат:** Сообщение об ошибке сети, возможность повторить

- **Ожидаемый результат:** Индикатор загрузки, обработка таймаута

```bash
# Создать новую БД
psql -U postgres -c "CREATE DATABASE telegram_seller_empty;"

# Изменить DB_NAME в .env на telegram_seller_empty
```

## Дополнительные рекомендации

### Проверка логов

Во время тестирования следите за логами в консоли, где запущен `npm run dev`. Обратите внимание на:
- Ошибки (красные сообщения)
- Предупреждения
- SQL запросы (если включено логирование)

### Проверка базы данных

Для проверки данных в БД можно использовать:

```bash
# Подключение к БД
psql -U postgres -d telegram_seller

# Просмотр таблиц
\dt

# Просмотр данных в таблице users
SELECT * FROM users;

# Просмотр данных в таблице shops
SELECT * FROM shops;

# Выход
\q
```

### Тестирование на разных устройствах

1. **Десктоп:** Chrome, Firefox, Edge
2. **Мобильные:** Chrome на Android, Safari на iOS
3. **Telegram Mini App:** Через реального бота в Telegram

### Резервное копирование

Перед тестированием рекомендуется сделать резервную копию БД:

```bash
# Создание резервной копии
pg_dump -U postgres telegram_seller > backup.sql

# Восстановление из резервной копии
psql -U postgres telegram_seller < backup.sql
```
